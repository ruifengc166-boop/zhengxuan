# 动态分镜文生图工作流实施计划

**目标：** 将固定镜头数和批量自动锁定改为动态镜头结构、文生图多轮抽卡和用户手动定稿。

**实现方式：** 在现有单页原型内重构分镜生产区，使用前端状态数组管理镜头总数、状态和定稿数；现有视频生产区读取动态总数。

**技术：** HTML、CSS、原生 JavaScript、Playwright 浏览器验收。

## 任务

- [x] 建立失败验收：动态工具栏不存在、固定 12 镜头仍存在、批量生成会自动锁定。
- [x] 重构分镜生产区的结构和视觉层级。
- [x] 增加动态镜头状态与新增、删除、拆分、合并、排序交互。
- [x] 改造批量 Prompt 和批量候选生成，禁止自动定稿。
- [x] 增加候选编辑、手动定稿和重新编辑失效提示。
- [x] 将视频阶段和剪辑提示改为读取动态镜头总数。
- [x] 完成浏览器端全流程验收和无报错检查。
